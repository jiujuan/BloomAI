# AI 画图（AI Image Studio）产品需求文档 PRD

> 版本：v1.0 · 状态：Draft · 日期：2026-07-01
> 归属模块：BloomAI 独立功能 —「AI 画图」
> 相关技术方案：[ai-image-studio-technical-design.md](./ai-image-studio-technical-design.md)

---

## 1. 概述

### 1.1 背景

BloomAI 现有 Chat 已具备文本对话、AI 写作、工具、Skills 等能力，后端 `generateImage()` 也已打通 OpenAI / Agnes 两条图像生成链路（见 [`src/server/llm/media/image.ts`](../../src/server/llm/media/image.ts)），但尚未有面向普通用户的「画图」入口——图像能力目前只作为内部工具（`image_gen`）存在，无独立页面、无参数化 UI、无模板复用。

本 PRD 定义一个**独立的一级功能「AI 画图」**：以 AI Agent 为核心，支持多种画图大模型，提供「比例 / 风格 / 参考图 / 模板」等参数化能力，并为未来通过第三方 Skills 画图预留扩展点。

### 1.2 产品定位

一句话：**面向创作者的「对话式 + 参数化」AI 出图工作台**。

- **对话式**：中间栏是一个 chat，用户用自然语言描述需求，Agent 可优化提示词、按需追问、并生成图片；出图结果直接展示在对话流里。
- **参数化**：输入框工具栏提供「模型 / 比例 / 风格 / 参考图 / 模板」下拉，降低 prompt 门槛。
- **模板驱动**：右栏是精选出图模板，一键「做同款」把模板提示词灌入输入框。

### 1.3 目标与非目标

**目标（v1.0）**

1. 独立「AI 画图」页面，左中右三栏布局。
2. 支持多画图模型选择（下拉切换），文生图 + 图生图（参考图）。
3. 参数化：比例（含语义标签）、风格（缩略图）、参考图上传、模板。
4. 出图结果在对话流内展示，支持下载 / 复制 / 做同款 / 再次生成 / 设为参考图。
5. 左栏画图会话（标题）列表，历史可回溯。
6. 右栏模板画廊，「做同款」把提示词填入输入框。
7. 会话与出图记录本地持久化（local-first）。

**非目标（v1.0，列入 Roadmap）**

- 局部重绘 / inpainting / 涂抹蒙版编辑器。
- 多图批量九宫格 upscale/variation 全流程（MJ 风格）——v1 仅单图，异步任务模型预留。
- 通过第三方 Skills 画图——v1 仅预留架构扩展点，不落地 UI。
- 团队协作 / 云端图库同步。

### 1.4 用户与场景

| 用户 | 场景 | 核心诉求 |
|------|------|----------|
| 内容创作者 | 公众号/小红书配图、封面 | 快速出图、比例贴合平台、风格统一 |
| 设计师 | 概念图、Moodboard、灵感探索 | 多模型对比、参考图控图 |
| 普通用户 | 头像、壁纸、表情包 | 模板一键做同款、零 prompt 门槛 |
| 开发者 | 本地模型（Ollama）出图 | 隐私、离线、可控 |

## 2. 功能描述

### 2.1 功能地图

```
AI 画图（独立一级页面）
├── 左栏：画图会话列表
│   ├── 新建会话
│   ├── 会话标题（自动命名 / 重命名 / 删除）
│   └── 会话切换（回溯历史出图）
├── 中栏：Chat + 出图区
│   ├── 对话消息流（用户 prompt / Agent 回复 / 出图卡片）
│   ├── 出图卡片（图片 + 参数徽标 + 操作：下载/复制/做同款/重绘/设为参考图/查看大图）
│   └── 输入区（Composer）
│       ├── 提示词输入框（多行）
│       ├── 工具栏 Chips：
│       │   ├── 画图模式标识（图像生成）
│       │   ├── 模型选择 ▾        （参考图 1 的 UI）
│       │   ├── 比例 ▾            （参考图 2 的 UI）
│       │   ├── 风格 ▾            （参考图 3 的 UI）
│       │   ├── 参考图 +（上传/粘贴，0~N 张）
│       │   └── 模板（打开右栏 / 快捷选择）
│       └── 生成按钮
└── 右栏：模板画廊
    ├── 分类筛选（全部 / 人像 / 风景 / 二次元 / 商业 / 壁纸 …）
    ├── 模板卡片（缩略图 + 标题 + 标签）
    └── 「做同款」按钮 → 提示词灌入中栏输入框
```

### 2.3 支持的画图模型（Model Catalog）

模型以「Provider → 模型」两级组织。下表标注 v1 的落地状态；未落地的通过技术方案中的 Provider 适配器逐步接入。

| Provider | 模型（示例） | 类型 | 参考图 | v1 状态 | 接入方式 |
|----------|------------|------|--------|---------|----------|
| Agnes（Sapiens AI） | `agnes-image-2.1-flash` | 文生图/图生图 | ✅ | ✅ 已打通 | 已有 `generateAgnesImage` |
| OpenAI | `dall-e-3` | 文生图 | ❌ | ✅ 已打通 | 已有 `generateOpenAIImage` |
| OpenAI | `gpt-image-1`（GPT Image） | 文生图/图生图 | ✅ | 🟡 适配 | OpenAI images 适配扩展 |
| Ollama（本地） | 本地图像模型（如 SDXL/Flux 系） | 文生图 | 视模型 | 🟡 适配 | Ollama 图像适配器 |
| Flux | `flux.1-pro / flux.1-dev` | 文生图/图生图 | ✅ | 🟡 适配 | OpenAI-compatible 图像网关（fal/replicate/openrouter） |
| Qwen | `qwen-image` | 文生图 | 视网关 | 🟡 适配 | OpenAI-compatible 图像网关 |
| Google Gemini | `nano-banana 2`（Gemini image） | 文生图/图生图 | ✅ | 🟡 适配 | Gemini 图像适配器 |
| Midjourney | `midjourney`（异步） | 文生图/图生图 | ✅ | 🟠 预留 | 异步任务 + 轮询（第三方代理网关） |

> 说明：参考截图中的 “Seedream 5.0 Lite / 4.5 / 4.0” 为 UI 交互参照样式。BloomAI 的实际模型目录以上表为准（Agnes / OpenAI / Ollama / Flux / Qwen / Gemini / Midjourney）。模型目录由数据库 `llm_models`（modality=image）驱动，可在设置中增删。

```

```

- **图像生成**：当前模式标识（预留未来「图像编辑」等模式切换）。

## 4. 交互流程与步骤

### 4.1 主流程：文生图（含模板做同款）

```
开始
 │
 ├─(A) 直接输入： 用户在输入框描述 → 选模型/比例/风格 → 点「生成」
 │
 └─(B) 模板做同款： 右栏点「做同款」 → 提示词+推荐参数回填输入框 → 微调 → 点「生成」
        │
        ▼
   [前端] 组装请求 { sessionId, prompt, model, aspectRatioId, styleId, referenceImages? }
        │
        ▼
   [中栏] 立即渲染：用户消息气泡 + 出图占位卡（loading）
        │
        ▼
   [后端] 解析参数：比例→尺寸、风格→提示词增强，路由到对应 Provider 适配器
        │
        ├── 同步 Provider（Agnes/OpenAI/…）：直接返回 url/b64
        └── 异步 Provider（Midjourney）：创建任务→前端轮询进度
        │
        ▼
   [后端] 下载图片落地本地（media 目录）→ 记录 image_generations
        │
        ▼
   [中栏] 占位卡替换为结果卡（图片 + 参数徽标 + 操作）
        │
        ▼
   [左栏] 首条生成后自动命名会话标题
   结束
```

### 4.2 子流程：图生图（参考图）

```
用户点「参考图 +」→ 选择/粘贴/拖拽图片
   │
   ▼
Composer 显示参考图缩略图（可删除，可多张）→ 模式自动切到「图生图」
   │
   ▼
用户输入「要改什么 / 要保留什么」→ 点「生成」
   │
   ▼
前端把参考图转为 Data URI（或已上传得到 URL）随请求发送
   │
   ▼
后端将参考图放入 Provider 的 image[] 参数（如 Agnes extra_body.image）
   │
   ▼
出图结果卡展示（可再次「设为参考图」形成迭代）
```

### 4.3 子流程：切换/管理会话

- 新建会话：左栏「＋」→ 清空中栏 → 新 session。
- 切换会话：点左栏条目 → 加载该 session 的历史消息与出图。
- 重命名/删除：条目 hover 出操作（复用现有 SessionList 交互）。

### 4.4 异常与边界

| 场景 | 处理 |
|------|------|
| 未配置 API Key | 模型置灰；点击提示「前往设置配置 Key」，可跳转设置页 |
| 生成失败（超时/4xx/5xx） | 结果卡显示错误信息 + 「重试」按钮；错误来自 Provider 时透传 message |
| 参考图过大/格式不符 | 前端校验（大小/类型），超限压缩或提示 |
| 空提示词点生成 | 生成按钮禁用 |
| 生成中切换会话 | 生成任务与会话绑定，返回时仍可见结果/进度 |
| 异步任务长耗时 | 占位卡显示进度百分比；支持后台继续、切走再回来查看 |
| 离线/本地模型不可用（Ollama 未启动） | 明确提示本地服务未就绪 |

## 5. 数据与持久化（产品视角）

- **画图会话**：标题、创建/更新时间、默认模型。
- **消息**：用户提示词消息、Agent 文本消息、出图消息（引用一条出图记录）。
- **出图记录**：提示词、模型、比例、风格、参考图引用、结果（本地文件路径 / url / b64）、状态、耗时。
- **图片文件**：落地到应用数据目录的图库子目录（local-first，可在设置查看/清理）。
- **模板**：内置 seed 数据；结构见技术方案。

## 6. 验收标准（v1.0）

1. 侧边栏出现「AI 画图」入口，点击进入三栏页面。
2. 可切换至少 2 个已打通模型（Agnes / DALL·E 3）成功出图。
3. 选择比例后出图尺寸随之变化；选择风格后出图风格明显变化。
4. 上传参考图可完成一次图生图（Agnes）。
5. 出图卡支持下载 / 复制 / 查看大图 / 重绘 / 做同款 / 设为参考图。
6. 右栏模板「做同款」能把提示词填入输入框并可直接生成。
7. 新建/切换/删除会话正常；刷新应用后历史出图仍在。
8. 未配置 Key、生成失败等异常有明确提示且不崩溃。

## 8. 已确认决策（Resolved Decisions）

1. **图片默认落地目录**：默认落地到**应用数据目录**（`<appData>/bloomai/images/…`）；设置页提供「图库目录」可修改，同时可查看/清理。
2. **参考图上传**：v1 先做**本地 Data URI 直传 Provider**（Agnes 等支持 Data URI）；未来需要公网 URL 的 Provider（如 Midjourney 类）再补「先上传得 URL」链路。
3. **数据存储**：采用**方案 A** —— 新增 `image_sessions` 表，**复用 `messages`** 存对话与出图卡，画图会话与聊天会话解耦（见技术方案 §5.4）。
4. **Agent 提示词优化**：**默认开启**；Composer 提供一个可选开关（「智能优化提示词」），用户可关闭以原文出图。
