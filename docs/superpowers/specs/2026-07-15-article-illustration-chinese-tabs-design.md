# 文章配图页面中文化与标签页设计

**日期：** 2026-07-15
**范围：** `ArticleIllustrationWorkbench` 的前端展示与交互，以及 Store 对当前可视来源的选择；不改后端接口或文章配图任务数据结构。

## 目标

将“文章配图”工作台改为中文界面，并让可用内容来源和生成路由通过清晰的标签页进行选择；整体视觉与既有“AI 画图”工作台保持一致。

## 页面结构与中文文案

页面标题、说明、字段、按钮、状态与错误信息均改为中文。例如：

- `Article illustration` → `文章配图`
- `Choose a source, review the editable scene plan, then confirm generation.` → `选择文章来源，编辑配图方案后确认生成。`
- `Paste article text` → `文章文本`
- `Upload MD, TXT, DOCX, PDF` → `上传文件`
- `Generation route` → `生成方式`
- `Package Skill (preferred)` → `Skill 优先`
- `Current image model fallback` → `现有模型兜底`

## 文章来源标签页

在来源卡片标题和说明下方增加两个 `role="tab"` 标签：

1. **文章文本**：显示正文 `textarea`。
2. **上传文件**：显示 `.md/.markdown/.txt/.docx/.pdf` 文件选择控件与已选文件名。

“文章 URL”标签页及 URL 输入、服务端抓取授权复选框在本次改造中暂时不展示。URL 后端能力和已有数据结构不删除，但页面不再提供 URL 来源。

标签按钮拥有 `role="tab"`、`aria-selected`、`aria-controls` 和关联的 `role="tabpanel"`，键盘与读屏软件可辨识当前内容区域。

### 来源数据与校验

- 切换来源标签时只更改当前视图和当前校验范围，**不清空**已有正文或文件选择。
- Store 新增当前来源模式（`text` / `file`）。创建方案时只将当前标签页的数据提交给现有服务端校验：文本标签提交正文，上传文件标签提交文件路径和文件名。
- 在某一标签页实际输入正文或成功上传文件时，调用现有 `setSource` 规则清空另一来源；上传成功前保留原有内容。
- 初始来源模式为文章文本；成功上传文件后切换到上传文件标签。

## 生成方式标签页

将两个 radio 选项替换为一组 `role="tab"`：

1. **Skill 优先**：调用现有 `setExecution('skill', ...)`，展示合规 Skill 下拉选择和现有授权/预算提示。
2. **现有模型兜底**：调用现有 `setExecution('fallback')`，隐藏 Skill 下拉选择，保留模型、数量、比例、风格等现有通用配置。

切换生成方式不会清除已选择的 Skill；切回 Skill 优先时恢复该选择。创建方案的禁用规则保持不变：Skill 优先但未选择可用 Skill 时不可提交。

## 视觉规范

- 继续使用 `image-studio image-studio-article` 容器与现有主题变量。
- 来源和生成方式采用与“AI 画图”一致的卡片边框、背景、圆角、间距、按钮与表单控件色彩。
- 新增通用 `.article-tabs` 样式：紧凑的横向标签条，选中态使用项目现有的 `--bg-info` / `--text-info` 主题色，未选中态沿用二级背景与边框；小屏可换行。
- 文本框、输入框、选择框全部使用现有 Image Studio 一致的边框、背景、文字颜色、焦点状态和尺寸；不保留浏览器默认的白底/黑框外观。
- 在窄屏下来源和生成方式标签可换行，但内容区域与现有单列响应式行为一致。

## 非目标

- 不删除既有 URL 抓取后端能力或 URL 数据字段；本次仅暂时移除其页面入口。
- 不改变文件解析、Skill 运行或模型兜底的业务行为。
- 不新增文件格式或后端校验接口。
- 不改变“AI 画图”单图工作台的布局和工作流。

## 验收标准

1. 页面不再显示英文标题、来源、生成路由、主要操作文案以及文章 URL 输入控件。
2. 文章文本、上传文件以两个标签页呈现；切换标签不会清空此前内容。
3. 创建方案只校验当前标签页的数据；仅在用户实际编辑、上传后清理另一来源。
4. Skill 优先与现有模型兜底以两个标签页呈现，并保留原有生成策略和禁用条件。
5. 新标签、文本框、输入框与“AI 画图”页面使用一致的主题化视觉样式，并具备无障碍 Tab 语义。
6. 相关组件和 Store 测试、类型检查和生产构建通过。
